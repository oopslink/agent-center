## 1. Flagship: Plan Orchestration

**Vision** (@oopslink): a PM agent (or human) groups pre-assigned tasks into a
**Plan**, lays out a **DAG** of dependencies, and the system **auto-advances** the
DAG — when a node's upstream is done it **@mentions the node's assignee in the
Plan's conversation** to pull them in (@human = notify, @agent = wake to do it).

**Full design + all decisions**:
[`docs/design/v2.9-plan-orchestration.md`](../design/v2.9-plan-orchestration.md)
(commit 20c3f55). It is authoritative — read §2 (model), §3 (lifecycle), §9 (the
8 hardened definitions), §6 (phasing), §8 (acceptance).

**Model in one diagram**:
```
Project → Plan (parallel; v3.0 & v4.0 are two Plans)
           ├── 1:1 Conversation (auto-created; creator + assignees auto-added)
           └── selects N backlog Tasks → depends_on DAG (acyclic, 1:1 with Plan)
```

**Confirmed decisions** (don't re-litigate — @oopslink locked these):
1. **Fully autonomous, no approval gates.** Node fails → notify the Plan creator;
   if creator is an agent it self-handles (adjust DAG / escalate), else human.
2. **Manual assignment** (PM agent / human pre-assigns) — no auto skill-matching.
3. **Dispatch = @mention in the Plan conversation** (reuses wake+mention #220);
   no new dispatch channel.
4. **Advancement trigger = work-item status → done** (assignee-owned, as today);
   no auto completion detection.
5. **No Stage layer in v1.**
6. See design §9 for: Plan done⟺all-nodes-done; Task.status vs derived node
   status; dispatch idempotency; no-DAG-edit-while-running; additive participant
   sync; start validation; failure isolates only the dependent subtree;
   Plan↔DAG 1:1.

## 3. Phasing (full feature)
- **P1** (this batch): Plan + Plan-conversation + task↔plan + DAG + DAG view +
  **manual advance**.
- **P2**: the **orchestrator** — event-driven auto-advance on status→done +
  @mention dispatch + failure→creator notification.
- **P3**: **PM-agent MCP authoring** — `create_plan` / `add_task` /
  `add_dependency` / `start_plan` so a PM agent drafts the plan + DAG from a
  discussion; human edits in the UI.
