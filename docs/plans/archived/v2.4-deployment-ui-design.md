## § 1 设计目标

服务于 v2.4 主线诉求："让 @oopslink **首次在 Web Console 加 worker** 是丝滑的、产品级的"。

具体 UX 目标：

| 目标 | 衡量 |
|---|---|
| **零文档可完成** | 用户不读 docs / readme 就能把第一个 worker 加上线 |
| **2 分钟内** | 从点 "Add Worker" 到 WC 看到 worker 上线 ≤ 2 分钟（含切机器 + 粘贴执行） |
| **失败可恢复** | 任何失败状态都告诉用户"接下来按什么" |
| **不打扰** | 不引入新页面、不破坏现有 `/fleet` 信息层级 |

## § 3 主用户流（happy path）

```
┌─────────────────────────────────────────────────────────────────────┐
│ 用户安装 center 完成，浏览器打开 http://127.0.0.1:7100               │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 落地 Home 页 — 看到 "Get started: add your first worker" 卡片         │
│ （或落地 /fleet — 看到空状态 + 大 CTA "Add a worker to start"）      │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 点 CTA → Add Worker Modal 弹出，State 1：Ready                       │
│ - 显示一条 install 命令（含 token + fingerprint 预填）               │
│ - Copy 按钮                                                          │
│ - "Token expires at 9:47 AM (in 28 min)"                            │
│ - "⏳ Waiting for worker..."                                         │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ 用户切到目标机器（mac / linux），cd 到已解压 tarball 目录            │
│ 粘贴执行命令 → worker daemon 启动 → bootstrap 到 center            │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│ Center 收到 worker enroll 事件 → 通过 SSE 推送到 WC                  │
│ Modal 翻到 State 3：Success                                          │
│ - "✓ Worker 'mbp-1' connected!"                                      │
│ - 显示 worker 信息（name / host / OS / version）                     │
│ - 按钮：[Add another worker]   [Done]                                │
│ - 背景 `/fleet` 表格新增 worker 行，3 秒淡出高亮                     │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
                  ┌─────────────────────────────────┐
                  │ 用户点 [Done] → Modal 关          │
                  │ 用户在 /fleet 看到新 worker 列在表里 │
                  │ 第一公里完成 ✓                    │
                  └─────────────────────────────────┘
```

## § 4 Add Worker Modal 状态机

> **v2.5 update**: this section is **archived**. The Modal collapsed
> to a 3-state machine in v2.5-F1: `name_prompt` / `minting` /
> `mint_error`. The Modal closes immediately on mint success and
> the install command + wait-for-daemon UI moves to the Fleet row
> (Show install command / Re-mint install command actions, F2 #54).
> States 1 (Ready), 3 (Success), and 6 (Timeout-hint) below are
> retired — the design rationale here documents v2.4 intent only.
> See `docs/deployment/v2.4-first-mile.md § 3.1` for the v2.5 flow.

### § 4.1 状态图

```
        ┌─────────────┐
        │ State 0     │
        │ Minting     │  ← 用户点 "Add Worker"，token 还没 mint 出来
        │ (<1s)       │     此状态短暂，看到的概率很低
        └──────┬──────┘
               ↓ token minted
        ┌─────────────┐
        │ State 1     │
        │ Ready       │  ← 命令显示，等待 worker 连接（主状态）
        │ (常驻)      │     spinner 转，倒计时 token 过期
        └──┬──────┬──┬┘
           │      │  │
           │      │  └────────────────────────────────────┐
           │      │                                       │
           │      └─→ State 5: Token expired              │
           │      ↑   "Token expired"                     │
           │      │   [Generate new token] → 回 State 1   │
           │      │                                       │
           │      └─→ State 4: Token used                 │
           │          "Token already used by Worker X"     │
           │          [Generate new token] → 回 State 1   │
           │                                              │
           ↓ worker connected                             │
        ┌─────────────┐                                   │
        │ State 3     │                                   │
        │ Success     │  ← worker 连上了                  │
        │             │  [Add another worker] → 回 State 0│
        │             │  [Done] → Modal 关                │
        └─────────────┘                                   │
                                                          │
        ┌─────────────┐                                   │
        │ State 6     │  ← 5 分钟没动静（既没连上也没过期）  │
        │ Timeout-hint │  软提示，不阻塞                   │
        │             │  "Worker hasn't connected yet."   │
        │             │  显示 troubleshooting 提示         │
        └─────────────┘ ←─────────────────────────────────┘
```

注：State 2（Connecting）作为 worker mid-handshake 的可选过渡态被合并 — TCP enroll 实际几百毫秒，不显单独 State，UI 不视为独立状态。

## § 6 文案 tone

### § 6.1 原则

- **直白 + 友好**：不卖萌（无 emoji ✨🎉），但不冷漠
- **第二人称**："Your worker"、"On your machine" 而不是 "The user's worker"
- **行动导向**：错误文案永远以"接下来做什么"结尾，不只描述现象
- **技术词保留英文**：Worker / Token / Fingerprint / Center / Tarball 等不强行汉化
- **中英文混排**：默认 UI 字符串英文（与现有 WC 风格一致），文档 + 内嵌帮助可以中文（看现有惯例）

### § 6.2 反例

❌ 「Token 已失效，请重试」(why? what to do?)
❌ 「Oops! Something went wrong 😢」(playful but useless)
❌ 「The token associated with this enrollment session has expired due to elapsed time exceeding the 30-minute window.」(too formal)

### § 6.3 正例

✅ 「Token expired (30 min cap). Generate a new one to keep going.」
✅ 「Couldn't reach center at home.lan:7300. Check network or firewall.」
✅ 「Worker 'mbp-1' is already enrolled. Choose a different name or remove the existing one first.」

## § 7 SSE 实时反馈 wire-up

UI 行为依赖以下事件（订阅信号需 wire 到 SSE bus，假定后端已有或好补）：

| 事件类型 | UI 触发 |
|---|---|
| `worker.enrolled`（含 worker projection）| Modal State 1 → State 3；`/fleet` 表新增 + 高亮 |
| `worker.enroll_attempt_failed`（含 reason）| Modal State 1 → State 4（token used）/ State 5（expired）/ stay-with-warning |
| `admintoken.expired`（按 token id 过滤） | Modal State 1 → State 5 |
| 5 分钟无任何前述事件（前端 timer） | Modal State 1 → State 6（soft hint，不阻塞）|

UI 不长轮询；不刷新；不重连（除非 SSE 断线，那是另一回事 — 既有 SSE 重连逻辑覆盖）。

## § 8 边界 & 失败 UI 矩阵

承接 [deployment doc § 5](v2.4-deployment-first-mile.md#-5-失败--恢复矩阵)，UI 层映射：

| 失败场景 | UI 表现 | recovery action |
|---|---|---|
| Center 端口被占（用户在 CLI 看到，不在 WC）| 不在 UI 范围 | CLI 文案，[deployment doc § 3.1](v2.4-deployment-first-mile.md) |
| Worker 跑命令 → center 不可达 | Modal State 1 持续 + 5 分钟后 State 6 hint | 用户检查网络 / 防火墙 |
| Token 已 burn | Modal State 4 | Generate new |
| Token 过期 | Modal State 5 | Generate new |
| Fingerprint 不匹配 | worker 端报错（CLI），WC Modal 无变化（不会触发 enroll 事件）| 用户对比 Modal 上的 fingerprint 摘要 + worker 端 |
| 重复 worker name | worker 端报错（"Worker 'mbp-1' already enrolled"），WC Modal 无变化 | 用户改 name 或先移除 |
| SSE 断线（WC 后端临时挂） | 既有 SSE 重连逻辑 + Modal 内"Reconnecting…" 微提示 | 等自动重连 |
| 用户关 Modal 但未用 token | 后台 revoke 该 token | 透明，下次重开重 mint |

## § 9 小决策清单（待 @oopslink 拍）

| # | 决策 | 推荐 | 备选 |
|---|---|---|---|
| **D1** | Home 页（`/`）也要加"Get started: add your first worker"卡片吗？| **是** —— Home 是第一次打开 WC 的落地页，empty state 应该引导 | 否，只在 `/fleet` 引导，Home 保持通用 |
| **D2** | Modal 关闭 + 未用 token → 自动 revoke 吗？| **是** —— 收缩攻击面，token 不留遗留 | 否，留着等过期；可能被忘的 token 是风险 |
| **D3** | Token expire countdown — 显示具体时间 vs. 秒级倒计时？| **混合**："Expires at 9:47 AM (28 min)" + 每分钟更新 | 只数字倒计时 |
| **D4** | "Add another worker" inline action vs 关 Modal 重开？| **inline** —— 顺手加多个 worker，减少摩擦 | 关 Modal 重开 + 关-mint-显示 3 步 |
| **D5** | Worker 上线 toast 默认时长？| **5 秒** 自动淡出 + 点关 | 持续不淡出，用户必须手关 |
| **D6** | Modal 用户多 tab 同时打开？| 每 tab 独立 mint 自己的 token（互不影响）| 全 session 共享 token；UI 复杂度高 |
| **D7** | `/fleet` 顶栏 "Add Worker" 按钮颜色 — primary（醒目）vs secondary？| **primary**（站第一公里阶段）→ 等到有几个 worker 之后可视情降为 secondary | 一直 primary |

## § 10 非目标（v2.4 first-mile 不做）

- **Worker 详情页改造** —— `/agents/<id>` 页（已有）不在本期 scope
- **批量 worker 操作**（一次加 N 台）—— 单 worker enroll 流程足够，多机 → 重复多次（D4 inline 已减少摩擦）
- **Worker remove / drain / quarantine UI** —— 既有 CLI 接口；v2.4 first-mile 焦点是 **加**，不是管
- **Mobile / responsive** —— WC 桌面专用
- **CLI install 命令的输出格式美化** —— 不在 WC 设计范围；归 CLI UX，hand-off 给 @AgentCenterDev 在 CLI 层处理
- **Visual / code 实现** —— 本文档止于交互，hand-off 给前端实现
