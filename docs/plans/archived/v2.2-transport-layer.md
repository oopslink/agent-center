# v2.2 Transport Layer — overall plan

> **Why this plan exists**: v2.0 GA shipped with CLI / worker daemon /
> server all directly opening the same sqlite file (verified by
> @oopslink firsthand 2026-05-24). This violates conventions § 0.4
> (AppService is the only entry to domain state). v2.2 rebuilds the
> transport layer so the server process is the single sqlite writer
> and every other process talks to it through the AppService surface.

## § 0. Domain-first scope (per conventions § 0.4 必答铁律)

| 必答问题 | 答 |
|---|---|
| 起点是不是"领域模型要求什么"？ | 是——DDD 要求 AppService 是唯一入口；single-host vs multi-host 是 transport 选择，不动 AppService |
| 是否保留 BC invariants？ | 是——server 是唯一写入者，所有 version CAS / same-tx event emission / 状态机过渡在 server-side AppService 内完成 |
| 是否绕过 transport？ | 否——unix socket 是 transport（不是"省 transport"），同协议可换 TCP 走 multi-host |
| Mock-as-default 是否消除？ | 是——dispatch.NoopSender / kill.NoopKillSender / nil SupervisorSpawner 全部替换为真实现 |

## § 1. Phase 拆分 (5 个 slock task)

5 个 phase 串行，每个 phase 用 per-ST P12 cadence（audit-first + impl + verify）。

| Phase | Slock task | 范围 | ETA |
|---|---|---|---|
| **A** | #13 | Server-as-SoR — 加 unix socket 监听 + admin endpoint，AppService surface 暴露 | ~5d |
| **B** | #14 | CLI refactor — `Client` 类型 + 所有 `handlers_*.go` 改成 client over transport | ~5d |
| **C** | #15 | `cmd/worker-daemon` binary — bootstrap-token exchange + dispatch stream + subprocess spawn | ~5d |
| **D** | #16 | Deployed-binary e2e — server + worker daemon + fakeagent，LLM-independent | ~2d |
| **E** | #17 | Process gates — arch test + mock-as-default lint + doc-impl drift + test report layered + deploy-smoke gate | ~2.5d |

总 ~19.5d（约 3-4 周）。

## § 2. Phase A 详细子分解（执行中）

### A1: admin endpoint scaffolding — unix socket binding

- 新增 `internal/admin/api/` 包（与 `internal/webconsole/api/` 平行）
- 复用 `HandlerDeps` 模式（直接复用现有 deps 不重新发明）
- `ServerCommand` 起服时同时 bind:
  - WebConsole TCP `127.0.0.1:7100`（现有）
  - Admin unix socket `/tmp/agent-center.sock`（新增；config 可改路径）
- 权限：socket file mode 0600 + owner = current uid（同机 trust，无 token）
- ETA: 1d

### A2: AppService surface mapping — admin endpoint coverage

把每个现有 CLI 命令所调用的 AppService 方法都映射到 admin endpoint。
策略：大部分 CLI 用的 Service 已被 WebConsole 暴露——admin endpoint **复用 webconsole/api handlers**，只加 CLI 独有的端点（worker enroll、dispatch、supervisor commands、project CRUD、identity registration 等）。

ETA: 1d

### A3: 真 EnvelopeSender — 替换 `dispatch.NoopSender{}`

- 新类型 `dispatch.InProcessQueueSender`（v2.2 single-host 版本）：往 server 进程内的 channel 推 envelope，worker daemon 通过 admin endpoint stream 拉取
- `kill.InProcessKillSender` 同样
- `// FIXME(prod-wiring): no real transport` 从 `app.go:167` 删除
- ETA: 1d

### A4: SupervisorSpawner 真 wire + dispatch reconciler

- `ServerCommand` 里 `app.SetSupervisorSpawner(scheduler.NewSpawner(...))` — 用真 ExecProcessRunner
- 起一个 dispatch reconciler goroutine 监听 envelope queue + 维护 worker liveness
- ETA: 0.5d

### A5: arch test — `persistence.Open` 白名单

- 新 test `internal/cli/arch_test.go` — grep `persistence.Open` in `handlers_*.go`，白名单：`handlers_migrate*.go` (schema 迁移) + `handlers_system.go` (server boot)
- 失败 message 引到 conventions § 0.4
- ETA: 0.5d

**Phase A 自验收**: A1-A5 全 green + 临时 deploy smoke (server 起来 + curl unix socket health) — 不依赖 phase E 的 gate。

## § 3. 不在 v2.2 scope 的事

- **multi-host TCP transport**: ADR-0040 已写 admin endpoint 支持 unix socket / TCP 双 transport，但 v2.2 只启 unix socket；TCP 启用 + bootstrap-token auth + 跨机部署留 v2.3
- **WebConsole 重写**: 现有 WebConsole HTTP API 不动；它已经在走 AppService（这部分本来就对的）
- **v1→v2.2 数据迁移**: schema 不动（admin 是 process 层 transport，不改 DB schema），现有 migrate tool 继续用

## § 4. ADR 需新增/修正

- 新 **ADR-0040** "transport-as-DDD-port"——AppService 是 port，transport（unix socket / TCP）是 adapter
- 修 **ADR-0001** § 24 行措辞——"CLI 自动检测 transport" 现在真实成立
- 修 **ADR-0038** § 88 行——admin endpoint 真存在，不再是"应该有"
- 修 `docs/design/architecture/tactical/presentation/01-web-console.md` § 78——"CLI / Web Console 两个独立的前端" 改成实际描述（CLI 走 admin endpoint，WebConsole 走 webconsole endpoint，两者都走 server-side AppService）

## § 5. 风险

- **CLI test 大幅重写**: 现在大量 CLI test 直接 setup DB + 调 Service。改成 client over transport 后要起 in-process server。是 B 阶段的工作量大头。
- **dispatch transport semantics**: 是 push (server → worker) 还是 pull (worker → server poll)？决策：**worker 长连接 stream**——server 维护每 worker 的 envelope channel；worker daemon HTTP long-polling 或 SSE。复用现有 SSE 基础设施。
- **e2e 验证不依赖 LLM**: 用 `cmd/fakeagent`（已存在 Phase 7 测试 harness）。Phase D 详细。
- **Backward compat**: v2.2 是 breaking change（CLI 行为：不连 server 直接报错）。需要在 v2.2 release notes 明示，并在 ServerCommand 启动时打 banner 提醒首次用户。

## § 6. 验收（v2.2 整体）

- 所有 5 个 phase task done
- `persistence.Open` 在 `handlers_*.go` 出现 ≤ 2 处（migrate + server boot 白名单）
- `dispatch.NoopSender` / `kill.NoopKillSender` 从 production wiring 消失（仍可作为 test 用 stub）
- e2e 跑通 `tests/e2e/v2/tests/deployed-task-pipeline.spec.ts`（server + worker daemon + fakeagent）
- conventions § 0.4 5 个 enforce 机制全装上
- 文档对齐（ADR + DEPLOYMENT + web-console.md）
