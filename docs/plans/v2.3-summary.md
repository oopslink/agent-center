## § 2 Task 详情

### #24 v2.3-1 Transport completeness
**领域诉求**：CLI / worker daemon / Web 前端访问领域状态**只能**通过 AppService（conventions § 0.4 铁律）；v2.2 留了 4 个 path 没接 admin endpoint。

**落地**：
- `POST /admin/conversation/participant/leave`
- `GET /admin/conversation/msg/find-recent`
- `GET /admin/taskruntime/task/read-context`
- `POST /admin/workforce/worker/heartbeat`（升级：v2.2 "enroll + swallow 409" hack → 真 200）

**副产出**：`WorkerEnrollService.Heartbeat` 服务方法、`TaskReadContext` Client 返回 `json.RawMessage`（线材稳定性）。

**新 gap**：无显式。dispatcher/sender wiring + MCP injection 早就在 #26-#27 待办里，不算 "由 #24 暴露"。

### #25 v2.3-2 ADR-0014 § 2 atomicity 复位
**领域诉求**：dispatch + DecisionRecord（kill + DecisionRecord）必须同事务（ADR-0014 § 2）；v2.2 transport 拆分后被打破。

**落地**：复合端点 `dispatch-with-decision` / `kill-with-decision`；transport 层用 `AGENT_CENTER_INVOCATION_ID` 显式区分 supervisor-driven（原子）vs worker/daemon-driven（简单）；DB 注入 `HandlerDeps` 做 per-request tx 边界；嵌套 `RunInTx` 重用（测试 `TestRunInTx_NestedReusesOuterTx` 证明）。

**副产出**：嵌套 tx 复用模式现成 — 后面再有"两步必须同事务"的端点照搬。

**新 gap**：Supervisor CLI 子命令（record-decision、spawn loop）真 e2e（fork subprocess + dispatch）当时跳过，挪到 #29 真 agent dispatch 链路接通顺手做。

### #28 v2.3-3a Admin token auth foundation
**领域诉求**：v2.0/v2.1 用 "loopback unix-socket = 隐式信任" 是 transport 工件不是认证；要给 #27 multi-host 和 #29 真 agent dispatch 打 auth 底座。

**落地**：新建 `admintoken` BC（AR/Repository/Service + 迁移 0028）；中间件挂所有 admin endpoint（白名单 `GET /admin/health`）；token CRUD endpoint；`bootstrap_token` 首启写入（owner=`system:bootstrap` / scope=`*`）。

**关键 design**：
- Plaintext 格式 `acat_<43字符 base64url>` —— 可 grep、prefix 进 hash
- Scope **不可变**（要改→ revoke + recreate）
- `MarkUsedAsync` pump（30s 节流）解决 macOS SQLite WAL 高频 UPDATE 时的 SQLITE_LOCKED race
- ADR-0026 "plaintext-never-echo" 扩展到 admin transport（list 不回 plaintext，事件不带 plaintext）

**副产出**：scope set v0（`*` / `admin:token` / `secret:resolve` / `blob:put` / `dispatch:pull` / `task:*`）；中间件 transport-agnostic（#27 只需换 listener，鉴权层不动）；测试 harness 每 setup 现造 token 不复用。

**新 gap（明确写在 audit § 5）**：
- TTL / expiration（当前 revoke-only，无到期）
- 审计事件（token create / revoke 不发事件，Service 已留接口）
- 跨 endpoint scope 细化（middleware 认 auth，scope 检查随 endpoint 落地；#29 落了 `secret:resolve` + `blob:put`，未来按需补）

### #29 v2.3-3b Real-agent dispatch chain
**领域诉求**：v2.2 ship 的 `defaultAgentSpawner` 走 fakeagent 短路；真 claude-code / codex / opencode adapter 需要 secret resolve + MCP 配置 + 工作目录三个外部资源。

**落地**：
- `/admin/secret/user-secret/resolve`（scope `secret:resolve`，base64-std plaintext echo）
- `/admin/blob/put`（scope `blob:put`，base64 + rel_path 防 OS escape）
- `DispatchEnvelope.HomeDir` 字段
- Worker daemon `AdminClient.ResolveSecret` / `BlobPut`；`ReportArtifact` 加 blob 前置
- Spawner 改写：fakeagent 路径不变；真 agent 走 `AssemblePrompt(loader, BaseSkill="worker-agent.md", HomeDir=env.HomeDir)` + `MCPInjector.Inject(env.HomeDir)`；失败 → `ReportFailure(reason="secret_unresolvable")` 立即结束
- Daemon `--skills-dir` flag；`FSSkillLoader` + `NewAdminClientSecretResolver` + `NewMCPInjector` 串起来

**副产出**：`adminClientSecretResolver` shim 维持 `MCPInjector.SecretResolver` 接口稳定；rel_path 验证防御深度（transport + service 双层）；HomeDir 空 = graceful degrade（v1 envelope 兼容）。

**新 gap（audit § 5 明确列）**：
- **AgentInstance.HomeDir 写时 config 字段** —— 当前是 convention path（`~/.agent-center-worker/agents/<id>/`），操作员**手工**保证目录存在 + 填 `instructions.md` / `mcp_config.json`；自动 scaffold 留给 v2.3-3c（**未开**）
- Real claude-code adapter e2e —— 需 LLM 或 VCR 录制
- BlobStore GET endpoint —— 本 ST 只 put（worker → center），read path 还走 server-local FS
- Multi-host TCP variant —— task #27
- master_key rotation flow —— 当前 audit doc 标 task #30，但 #30 实际被用 v2.3-4 占了；这条 rotation flow 还没建 task

### #31 v2.3-5a BC-native Issue/Task list endpoints
**领域诉求**：SPA Issues/Tasks 页之前调 `/api/conversations?kind=issue/task`，借 Conversation BC 当 Issue/Task 视图代理 —— **跨 BC 越界**（@AgentCenterDev 在 `#agent-center:97f6710d` 给出 4 option DDD 分析，@oopslink 在 `#agent-center:93496605` 选了 Option C 正确归位）。

**落地**：4 endpoints —— `GET /api/issues` + `GET /api/issues/{id}` + `GET /api/tasks` + `GET /api/tasks/{id}`；list 必填 `project_id`（缺 400），可选 `status`；projection 含 BC 内字段（issue: id/project_id/conversation_id/title/status/opened_at/opener/closed_at/closed_reason；task: id/project_id/conversation_id/title/status/priority/created_at/current_execution_id/depends_on_task_ids）。

**副产出**：`issuePublicMap` + `taskPublicMap` projection helper；BC 各自做权威（Discussion BC、TaskRuntime BC），Conversation BC 不再被借；coexist anti-regression（`/api/tasks/{id}` vs `/api/tasks/{id}/trace`）—— 防止未来路由收缩。

**§ 0.6 自检**：Issue projection 故意省 `kind`、`priority` —— 因为 Issue AR 不暴露这俩字段（priority 是 Task 概念）；不**伪造数据**。

**新 gap**：未来 SPA 想要 issue kind → 要 Discussion BC AR 扩展 + 迁移（不是 endpoint 改一改能搞定的）。

### #32 v2.3-5b SPA Issues/Tasks 切 BC-native 数据源

**新 gap**：本 ST 关闭了 v2.3 周期的最大 BC 越界 —— **无新 gap 产生**。

### #33 v2.3-6 Flaky CLI test fix
**领域诉求**：`TestClient_ConversationRead_TailUsesFindRecent_OverAdminEndpoint` 全包 25-30% 概率 FAIL（macOS only）—— 不是 prod bug 但卡 CI。

**根因**：v2.3-3a 引入的 `MarkUsedAsync` pump（30s 节流）在测试发 4 条消息时与 admin token 写竞争，触发 modernc.org/sqlite 的 `SQLITE_BUSY` 立即返回（不遵守 `busy_timeout=5000`）。

**修法**：`setupAdminServerForTests` 在 mint token 后立即 `app.AdminTokenSvc.Close()`；pump channel = nil → `MarkUsedAsync` 早返回 → 测试体内不再有 UPDATE 竞争。**仅 test 侧改动，prod 行为不变**。

**§ 0.6 严守**：audit § 5 明确把 "driver 是不是有 bug" "设计是不是要序列化" 这种归类为**推断**，不写进 audit。

**新 gap**：无。承认未来若有 test 在 ≤10ms 内发 ≥2 个 admin write 可能复现类似 flake —— 测试基建可考虑 hardening，但不是必做。

### #27 v2.3-4 Multi-host TCP transport — **in-flight**

**当前状态**：方案已落 `#agent-center:b79f992f`，**等 @oopslink 确认 5 个安全默认 + ST 拆法**。**未开始 coding**。

**领域诉求**：admin endpoint 当前只 listen unix socket（`unix:/path/admin.sock`，0600）→ server/worker/CLI 必须**同机+同用户**。worker daemon 设计上支持分布到多机跑 agent，但 transport 锁死了。

**待 @oopslink 决策的 5 条安全默认**（AgentCenterDev 已给出推荐）：

| # | 决策 | AgentCenterDev 推荐 | 备选（v3 升级路径）|
|---|---|---|---|
| 1 | TLS 方案 | 自签 + client fingerprint pinning（类 SSH known_hosts）| mTLS / CA-signed |
| 2 | Token rotation | revoke + recreate（无 grace）| grace-period rollover |
| 3 | Audit log | 加 client_ip 字段进 observability event | 加 user-agent / cert fingerprint |
| 4 | ACL | 仅 token scope，无 IP/CIDR | IP allowlist + per-IP rate limit |
| 5 | Rate limit | per-token 简单令牌桶（如 100 req/min/token）| 分布式 / per-IP / adaptive |

**ST 拆法**：v2.3-7a transport 抽象 + TCP listener / v2.3-7b AdminClient + worker daemon target / v2.3-7c per-token rate limit + audit / v2.3-7d e2e + DEPLOYMENT 更新。每 ST 约 2-3h，**总 ~10h**。

**产品层备注（PM 视角）**：这条不是技术债，是部署形态的功能扩展。**是否做 / 何时做** 取决于真实部署目标：
- 单 Mac 自用 → 不紧急
- worker 推云 VM 跑 → 这是 blocker

我的建议：和 @AgentCenterDev 推荐对齐，**轻量自签 + fingerprint**（操作员手动管 fingerprint pinning），先把多机能力打通；mTLS/CA 是 v3 deployment-as-product 主题的范围。
