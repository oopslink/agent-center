修复方案

问题根因：EmbeddedConversationSidebar 折叠态渲染为独立的 w-9 竖条（36px），插在 chat box 和 metadata sidebar 之间，造成三列对齐问题。

方案描述改动范围
A: 内移 toggle（推荐） 折叠时不渲染独立 w-9 竖条。将 expand 按钮移入 conversation banner 行右侧（与 Following / maximize 同行），消除中间空隙。chat box 直接贴邻 metadata sidebar。 EmbeddedConversationSidebar + WorkItemConversation
B: 对齐 padding 保留 w-9 竖条，但给 metadata sidebar 加相同的 pt 使顶部与 chat banner 的 Following 行对齐。治标不治本——仍有 36px 空白列。 IssueDetail / TaskDetail layout
C: 合并侧边栏 将 metadata sidebar 合并为 EmbeddedConversationSidebar 的一个 tab（Details / Threads / Files），消除双 sidebar 结构。工程量最大但布局最干净。 ConversationSidebar + TaskDetailSidebar + IssueDetailSidebar

合并方案详解

移动端将 MobileWorkItemBar 与 Conversation Banner 合并为一个 unified context bar：

  • 标题压缩为一行 — back arrow + 单行 ellipsis 标题（~30px），替代原来的 breadcrumb + multi-line title + TASK badge（~120px）
  • T89 ID + TASK badge 下沉到 merged bar — 不丢失，作为 bar 最左侧的 pill
  • Actions 下拉菜单 — Following / Info / Edit 合并为一个 "Actions ▾" 按钮，展开为 44px 高的 action sheet 项
  • Maximize 独立 — ⤢ 按钮单独保留在 bar 右侧，全屏聊天是移动端最高频操作
  • 总计省出 ~110px（标题 ~70px + 双 bar 合并 ~44px）→ chat 多展示 4-5 行

DT-3 · Mobile

Plan Detail Mobile Actions

PlanDetail 移动端已有 "Actions ▾" dropdown 收纳生命周期按钮（T341），但 tab 按钮、maximize/compact toggle 和 dropdown 内的按钮触控目标偏小。

DT-3aP1
Plan tab 按钮触控目标偏小
Chat / DAG / Task List tab 使用 px-3 py-1 text-xs（~28px），移动端核心导航应 ≥44px。与 ConversationMobileTabs 保持一致。
PlanDetail.tsx:168-176 (TabButton)
DT-3bP2
Maximize / Compact toggle 28px
Chat maximize 和 DAG compact toggle 均为 h-7 w-7 = 28px。移动端拇指操作困难。提升至 h-10 w-10 (40px) 或加大 padding。
PlanDetail.tsx:189, 204
DT-3cP2
Actions dropdown 内按钮偏小
Advance / Stop / Edit / Start 按钮在 dropdown 内使用 py-1.5 text-xs（~28px）。Dropdown 内每项应 ≥44px 高度并全宽，匹配移动端 action sheet 模式。
PlanDetail.tsx:341-397
DT-3dP2
PlanDetailHeader 信息密度过高
Header 的 Plan name + status + failed + auto-advancing + creator + date + target_date 在移动端窄屏上 flex-wrap 成多行，视觉混乱。建议移动端折叠为 compact bar（类似 MobileWorkItemBar）。
PlanDetail.tsx:312-315
DT-3eP3
DAG 移动端交互优化
DAG 在移动端已有 stepper view（非 graph），但 node 卡片的 assign/unassign 按钮和 dependency editor 在窄屏上可能溢出。需验证 < 375px 下的布局。
