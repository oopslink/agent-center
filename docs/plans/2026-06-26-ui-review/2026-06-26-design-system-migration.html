06 · Migration

Implementation Plan

渐进式迁移——先改 token 层（一次切换全局色彩），再改组件层（圆角/阴影/字体），最后改页面特有样式。

Phase 1
Token Layer — 颜色/字体/阴影
index.css: 替换 :root + :root.dark 的全部 CSS variables（~40 个 token）
tailwind.config.js: 更新 fontFamily (Plus Jakarta Sans + IBM Plex Mono)、boxShadow、borderRadius
index.html: 添加 Google Fonts link
影响: 全局生效，一次 PR 切换整体色调。组件代码零改动（全用 token）。
Phase 2
Component Layer — 圆角/阴影/间距
AppLayout: Module rail bg → eggplant; sidebar nav item active → accent bg
MessageList: Own bubble bg → eggplant + white text; other bubble → bordered card
MessageComposer: Send button → eggplant bg
All cards: border-radius 从 rounded/rounded-lg 统一到 rounded-lg (12px)
Modals: shadow 升级到 shadow-modal; radius → 16px
Phase 3
Page Layer — 页面特有调整
Status chips: 全部 status 色彩从 Tailwind palette 迁移到 Hex-inspired functional colors
Chat bubble: chatuserbubble token #D1E3FF → #31263B + fixed light text
Code blocks: bg #003247 → eggplant-dark variant
codeHighlight.css: 重新校验 WCAG AA 对比度
Phase 4
QA — 对比度校验 + 截图回归
a11y.test.tsx: 更新 raw color guardrail regex（新 palette 不再有 red-* 但可能有其他需屏蔽的）
所有 status token: 逐个校验 WCAG AA 4.5:1 on new surfaces
E2E 截图: 对比主要页面 before/after 确认无回归
