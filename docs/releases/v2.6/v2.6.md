# Agent Center v2.6 Release Notes

**Release date:** 2026-05-28  
**Upgrade path:** v2.5.x → v2.6.0 (fresh schema migration; see migration guide)

---

## Highlights

### Identity BC — multi-tenant auth

v2.6 introduces a full Identity Bounded Context with organization-scoped
multi-tenancy. Every install now requires a first-boot signup that creates the
initial Identity + Organization. Subsequent access requires signin via a 6-digit
passcode; sessions are managed via HttpOnly JWT cookies (7-day TTL).

New web pages: `/signup`, `/signin`, `/me` (passcode change), `/org/settings`
(org delete), `/members/humans` (member list + role management),
`/members/agents` (agent member list), Org Switcher in the top bar,
Sign Out in the sidebar footer.

### Org-scoped URL routing

All web UI routes now live under `/organizations/{slug}/...`:

```
/organizations/my-org/channels
/organizations/my-org/fleet
/organizations/my-org/tasks/...
```

Existing bookmarks without the org prefix redirect automatically to the
first organization.

### Supervisor BC removed

The `supervisor` CLI subcommand tree, `supervisor_invocations` and
`decision_records` database tables, and all associated domain services are
removed. Task dispatch and kill continue to work via the simplified path
(no supervisor decision records required).

## Upgrade notes

### Schema: drop + recreate

Migrations 0033–0036 run automatically at server start. They:

1. Drop and recreate the `identities` table with the BC9 schema
   (`kind IN ('user','agent')`, `passcode_hash`, `account_status`, no `version`).
2. Create new BC9 tables: `organizations`, `members`, `invitations`.
3. Drop the old `supervisor_invocations` and `decision_records` tables.

Conversations, projects, tasks, issues, secrets, and worker data are
**not affected** by these migrations.

### Bookmark loss

All v2.5 web UI bookmarks (URLs without `/organizations/{slug}`) become
redirects. Users will land on the correct page after the redirect resolves
(the first org they belong to).

### Uninstall → re-install path (fresh v2.6 install)

For a completely clean v2.6 install:

```bash
agent-center uninstall center
rm -rf ~/.agent-center        # or /var/lib/agent-center on Linux
agent-center install center
```

On first browser visit you will see `/signup`. Complete signup to create your
first Identity and Organization.

## What's new in detail

**BE-1:** Identity AR, Organization AR, Member AR, Invitation AR — SQLite repos,
factory methods, validation, domain error types.

**BE-2:** `SignupService`, `SigninService`, `SignoutService`, `AuthService`
(JWT HS256 using `master_key_file`), `PasscodeChangeService`.

**BE-3:** `MemberRoleChangeService`, `MemberDisableService`,
`OrganizationLifecycleService` (soft-delete), `OrganizationCreateService`,
`OrganizationLockManager`.

**BE-4:** `AgentIdentityProvisionService` (provisions `agent:` Identity for
Agent Instances), `IdentityBCFacade` (read-only interface for downstream BCs).

**BE-5:** 15 new domain event types (`identity.registered`, `auth.signed_in`,
`auth.signed_out`, `identity.passcode_changed`, `organization.created`,
`organization.deleted`, `member.added`, `member.role_changed`,
`member.disabled`, `member.reenabled`, etc.).

**BE-6:** `organization_id` FK column added to `channels`, `projects`,
`workers`, `user_secrets`.

**BE-7:** `agent_instances.identity_id` FK + `organization_id`; `supervisor`
enum value removed from AgentInstance kind.

**BE-8:** `internal/conversation/identity/` subdomain removed; `MessageWriter`
validates sender via `IdentityBCFacade`; admin identity endpoints removed.

**BE-9:** Supervisor tables/services/CLI removed; migration 0036 drops
`supervisor_invocations` + `decision_records`.

**FE-1–FE-6:** Full auth UI, org management UI, members management UI,
org-scoped routing.

## Configuration reference

The `identity.default_user` config key is **removed**. All other config keys
are unchanged.

The `secret_management.master_key_file` key is now also used as the JWT
signing key (HMAC-SHA256). No configuration change is needed; the same
32-byte AES key functions as both the secret encryption key and the JWT
signing key.

---

## Known limitations

- **Display name change:** the `/me` page does not yet support changing
  display names (requires `IdentityDisplayNameChangeService`, deferred to a
  future release).
- **Invitation flow:** the `Invitation` AR is implemented but the web UI for
  invite-by-link is deferred to v2.7.
- **Org settings PATCH:** org name/slug/description editing in the UI is
  deferred to v2.7 (the backend `OrganizationUpdateService` is ready).
