## 说明 / 限制
- **G3 进度真实性:** progressbar 元素与 XHR `upload.onprogress` 接线真实存在且渲染;但 localhost 上几 MB 的 PUT 请求体上传近乎瞬时(我用 route 把 PUT 响应延迟 1.5s 仍只能捕到 `aria-valuenow=0` 帧再跳完成)。即「进度 UI 真在、真随上传字节驱动」已证;但**大文件慢网下的逐帧百分比推进未在本地复现**。若要更强证据:终验阶段对真大文件(接近 25MB)+ 限速跑一次,或对 `xhr.upload.onprogress` 加单测断言多帧。
- 合成 DnD/paste 用 `dispatchEvent` 注入(Chromium 1223 honor `DragEvent.dataTransfer` / `ClipboardEvent.clipboardData` 构造参数);非真鼠标拖拽/真系统剪贴板。终验可酌情补一次真 OS 级拖拽(成本高、收益边际)。
