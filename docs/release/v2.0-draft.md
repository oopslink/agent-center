# agent-center v2.0 — Release Notes (draft)

> Status: **draft** — finalised at P12 release-gate. Phase 8-11 已 ship；
> P12 (cleanup + release checklist) 启动后这文档变 v2.0 正式 changelog。

## TL;DR

v2 是相对 v1 的重大重构：拿掉飞书 vendor 集成 / 全套换 Conversation v2 unified
model / 新增 Web Console SPA / Worker enrollment 走 bootstrap-token 流。

```
v1 → v2 单 binary：架构 + 数据模型 + UI 三层全换。不向后兼容 — v2 是新
                  installation，v1 install 走 P12 migration guide 升级。
```

## 重大变化（breaking）

### Bridge BC 撤回（ADR-0031）

v1 的 `internal/bridge/*` 整个 BC + 飞书 WebSocket / vendor 集成 / Identity[kind=vendor]
全删。v2 仅 loopback Web Console 一个 UI 入口；agent 跟 supervisor 之间通过
internal conversation BC 通讯。

### Conversation v2 unified model（ADR-0032 / 0039）

`group_thread` → `channel`；DM / Issue / Task 全部统一进 Conversation 聚合根；
participants 进 JSON 字段；message.carry_over_ref 替代 v1 的 vendor_msg_ref。
SQL schema 大动 — 见 migrations 0020-0024 + 0025（v1 bridge feishu tables drop）。

### Identity refactor (ADR-0033)

4 kind → 3 kind（删 `vendor`，剩 `user` / `agent` / `system`）；
identity ID 加 `kind:id` 前缀（如 `user:hayang`, `agent:01HXX...`）。
任何旧 bare-id 在 boundary 都被 reject。

### Worker enrollment overhaul (Phase 8 § 3.3)

旧的 worker 自报 → 新的 bootstrap-token exchange 流：
1. center 颁发短期 token (`agent-center worker token issue`)
2. worker 用 token 调 `worker enroll`
3. token consumed → 永久 worker identity；token 不可重用

### CLI 命令树重组 + `--format` 统一

- 全 list/show 命令支持 `--format=table|json|text`（默认 table；text 一行一 ID 给 xargs）
- `agent-center help <command-or-topic>` 含 topic 索引（format / identity / exit-codes）
- 30+ 顶级命令按 6 组分类：Help & info / Resources / Runtime / Observability / Admin / Other

## 新增

### Web Console (Phase 11)

完整 React SPA 嵌入单 binary：
- 13 page 覆盖 channel / dm / issue / task / agent / secret / input-request / fleet
- 实时 SSE：subscribe-once-per-app + 指数退避重连 + Last-Event-ID 恢复
- CV4 派生 UI：任意 conversation 详情 → "Select messages" → "Open Issue / Open Task"
- 安全：loopback bind only (ADR-0037) + secret value 永不外泄 (ADR-0026 § 5)

### SecretManagement BC (Phase 8 § 3.7-3.8)

新 BC 处理 user secret：
- master-key AES-256 encrypted at rest
- create-then-never-display 模型；rotation = revoke + create new
- `SecretRef` VO 让 agent / supervisor 引用 secret 不暴露明文（ADR-0027）

### CarryOver + Derivation services (Phase 10 § 3.5-3.6)

`CarryOverService` (CV3) — 跨 conversation message reference + `MessageDerivationService` (CV4) — 从源 conversation 派生 issue/task。CLI `agent-center issue open --from-conversation=...` / Web Console 多选 UI 一致体验。

### Observability + Fleet view

`/api/fleet` 4-segment snapshot：workers / active executions / open IRs / pending issues + warnings 容错。Web Console `/fleet` page 实时显示。

## 升级路径

**v1 → v2 直升不可能** — schema / API / Identity ID 都不向后兼容。Phase 12 提供:

1. v1 stop → snapshot v1 sqlite
2. 跑 v1-to-v2 migration tool（P12 § X）转换 conversation/identity records
3. fresh-clone v2 → load migrated db → start

v1 install 在 v2 GA 后保持 read-only 1 个月（看 dashboard 不接新 task）；之后 fully decommission。
