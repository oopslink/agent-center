## 1. Cross-phase gates (apply to EVERY phase)

These gates apply to **every** phase release. No phase ships without each row green.

### 1.1 Methodology discipline (Tester / Tester2 both)
- [ ] **Computed-truth assertion** (§1.1): all UI assertions read `getComputedStyle` / rendered DOM / HTTP code / byte assertion — never class-name inference.
- [ ] **Run-real-usage on real install** (§1): every acceptance run uses a real `install` codepath (real test-instance from `#255` helper; for P2/P3 also `#257 --with-seed`), never hand-crafted config / SQL shortcut.
- [ ] **Real product default prefix** (§1.2): path/discovery/layout features must run under `~/.agent-center` (not `/tmp`-isolated) where applicable.
- [ ] **False-alarm rule-out** (§1.1): every red/failing observation triggers harness-error + environment-noise check before being reported as a finding (concrete root cause, not "probably noise").
- [ ] **Deployed-smoke (≥1 per phase)** (testing.md §2.3): one full acceptance pass against a real deployed build (not unit-test only).
- [ ] **T-1..T-9 discipline** (`acceptance-methodology.md`): real defaults / real product prefix / real chained tools / shared-instance protocol / evidence-in-tree / completeness sweep / regression sweep / CRUD-completeness.

### 1.2 Quality red-lines (lint-enforceable from day-0)
Per PR #168 §5.6/§5.7 — these gates are enforced in CI, not just in review:

| Red line | Mechanism (CI) | Owner |
|---|---|---|
| no raw ULID in chrome (#192) | extend existing zero-raw-id regex sweep test | Tester2 verify; Dev2 implement |
| no `dangerouslySetInnerHTML` on unsanitized SVG/HTML | custom ESLint rule | Tester2 verify |
| no native `<dialog>` / `window.confirm` | custom ESLint rule (no-restricted-syntax) | Tester2 verify |
| ARIA picker / combobox required props | `eslint-plugin-jsx-a11y` recommended ruleset + `role-has-required-aria-props` | Tester2 verify |
| link-unfurl SSRF guard (P2 if applicable) | backend test asserts internal IP rejected + private-IP block list | Tester verify |
| `Content-Disposition: attachment` on downloads | header assertion test | Tester verify |

CI gate: all of the above green on every PR; no merge without.

### 1.3 D2 hard-cutover discipline (no-middle-state enforcement)
- [ ] **Pre-cutover snapshot**: backup of old component still buildable on a tag before merge (so revert is `git checkout <tag> -- <paths>` if needed, not a guessed undo).
- [ ] **Post-cutover smoke**: production build runs on real install codepath with no half-state observable in the 4 surfaces (channel/DM/task-thread/issue-thread all on the new MessageList; old component fully removed).
- [ ] **Rollback plan** (in commit/PR description): explicit single-commit revert path verified in advance (PR description includes "to roll back: `git revert <merge sha>`").
- [ ] **No flag fallback** — there is no feature flag; cutover is binary; old code paths deleted in same merge as new code paths land.

### 1.4 E2E mandatory (Playwright real browser)
- [ ] E2E suite covers each phase's user-visible flows on a real browser against a real running instance.
- [ ] CI: E2E green required for merge.

### 1.5 Evidence-in-tree (§13 + §20 + §21)
- [ ] Every acceptance run produces an evidence report at `docs/release/evidence/v28-p<N>-<phase>-<run>-acceptance-report.md`.
- [ ] Screenshots / fixtures live in `docs/release/evidence/screenshots/` and are committed **in the same PR** as the corresponding finding.
- [ ] Pre-merge: IntegrationDev verifies `git show <branch-tip>:<doc>` content-delta matches PR description claims (not just file presence).
- [ ] Post-merge: same content-delta check against `main` to confirm landed state.

### 1.6 Cross-phase regression sweep (every phase + final)

## 3. Phase 2 — Side Surface 1.0 (#265, D4+R2)

### 3.1 What ships
- `<SideSurfaceHost>` + `<ArtifactPanel>` resizable right-side panel; `<ArtifactCard>` inline collapsed; tab strip for multi-artifact; history back.
- Dual-display: inline collapsed/expanded + side panel; non-exclusive (both can be open).
- Auto-promote thresholds: client default (code > 20 lines / JSON > 5 KB / doc > 200 lines) + per-org/conversation server override (configurable).
- Backend: `artifact` entity new schema (id ULID / kind / payload / version / message_id / conversation_id / created_at) + `artifact_versions` table; 3 endpoints (POST/GET/PATCH `/api/artifacts?org_slug=`); deeplink routing `/<surface>?artifact=<id>`.
- a11y: `aria-modal=false`, no focus steal, Esc closes, Tab traverses normally, full keyboard nav including Ctrl+arrow resize.

## 4. Phase 3 — #/@ picker (#266, #248 + #/@ 4-layer spec)

### 4.1 What ships
- Composer ARIA combobox picker (trigger on `@` or `#` literal); fuzzy-match candidate list; keyboard nav (↑↓/Enter/Tab/Esc); icon-typed agent vs human.
- Backend: ref-candidate resolver endpoint (prefix → candidates, scoped to current org via `?org_slug=`).
- Storage: structured ref token (kind + stable id + user-typed form preserved).
- Render: chip with friendly name; hover full id card; click navigates to detail; status chip for work-item refs (StatusChip + orgRefToken reuse); EntityRef (#192) for boundary.
- #/@ accepts: typed friendly name (`@Bot Nine`) OR org seq (`#T34`) OR hash id (`#channel-abc123`) — all parsed and rendered as same chip.
- Paste: smart auto-convert exact matches (Slack-style) + internal copy preserves rich refs (fidelity).
- `#254 identityRef(kind, id)` helper extracted early in P3 (prevents `agent:`/`user:` prefix drop — closes #240 root cause).
- wake-projector string-match backward-compat preserved (old messages still wake correctly).

### 4.2 Tester data/API gates (owner: Tester — fill in independent doc-PR)
_To be filled per the §902d2df5 preview:_
1. ref-resolver candidate accuracy (prefix → candidate list, 6 entity kinds, both identifier dimensions accepted)
2. cross-org no-leak + nested `?org_slug=` correctness
3. structured ref token "never drifts" (display_name change / org_seq stable / id permanent — ref still resolves)
4. paste fidelity round-trip (auto-convert + internal copy round-trip preserves refs)
5. wake-projector string-match backward-compat (old plain-text @ messages still wake)

### 4.3 Tester2 UI/UX gates (owner: Tester2 — fill in independent doc-PR)
