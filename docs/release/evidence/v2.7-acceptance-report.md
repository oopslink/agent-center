## 🔴🔴 FINDING-E（§6/§8 — macOS release-blocker，新发现）：agent supervisor 控制套接字路径超长 → agent 无法运行/受控
**现象**：claude-code agent Start 后，worker daemon 进入**无限重启循环**，泄漏孤儿 claude+supervisor 进程（单 agent 实测 12+ 且增长），agent 无法 Stop（卡 `stopping`）。
**worker err.log 实证**（重复刷屏）：
```
agent_controller: start agent=…: start session: supervisormanager: supervisor did not come up within 15s:
stat …/agents/<aid>/supervisor.sock: no such file or directory (will retry)
```
**根因（已定位+量化）**：supervisor 在 `cfg.HomeDir/supervisor.sock` bind unix socket（`internal/supervisormanager/manager.go:186`）。HomeDir 由 `internal/workerdaemon/agent_controller.go:1081` = `filepath.Join(AgentHomeBase, "workers", WorkerID, "agents", agentID)` 组成；而 `AgentHomeBase`（`run.go:256 agentHomeBase`）= `<sqliteDir>/agent-homes` = `~/.agent-center/workers/<wid>/var/agent-homes` —— **已是 worker 作用域**，第 1081 行又**重复拼了一次 `workers/<wid>`**（路径里 `workers/<wid>` 出现两次）。
- 实测 sock 全路径 = **143 字节**，超 **macOS `sockaddr_un.sun_path` 上限 104 字节** → bind 失败 → supervisor.sock 永不创建 → worker 15s 超时 → 重试 → 无限重启。
- agent home 内有 `supervisor.instance` + `claude.pid`，**独缺 `supervisor.sock`**（全 worker 子树 find 无）—— 与"路径超长 bind 失败"完全吻合。
**仅去重路径不够**：去掉重复 `workers/<wid>` = 119B（仍超）；再去 `agent-homes` = 107B（仍超 104）。ULID agent-id(26) + "supervisor.sock"(15) 决定了**必须用短专用路径**（如 `$TMPDIR/acsv-<hash>.sock`，与 HomeDir 解耦，instance/pid 仍留 HomeDir），才能稳定 <104。
**影响**：**macOS 上每个 install 都中招**（用户名 oopslink 仅 8 字符、worker-id 标准长度即已 143）。§6 生命周期控制 + §8 真实受控执行**在 macOS 全断**。CI 多在短 /tmp 路径或 Linux(108) 下跑，故漏网——正是"按真实 install 路径在真实 OS 上验"才抓到。
**伴生问题（次要）**：
