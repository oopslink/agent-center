### 2.2 Headline: @oopslink's "multiple agents running simultaneously" bug ‚Äî eradicated, FOUR-FOLDÂùêÂÆû

The single-active invariant (one agent ‚áí at most one active WorkItem) is proven across four independent layers:

1. **Code race-gate** ‚Äî PR1 race-gate + migration 0051 single-active index.
2. **Real-LLM behavioral** ‚Äî a real claude-opus agent on a `--with-agent` install ran the full pull-loop (assign‚Üíwake‚Üí`start_work`‚Üíactive‚ÜíÁúüÂÅöÊ¥ª[ran `ls -A`, described output]‚Üícomplete); active-slot released to 0 (Tester2 report #3).
3. **Go-mechanism (deterministic)** ‚Äî PR7 `TestPR7_SingleActiveByConstruction`: an agent with two queued WorkItems has its 2nd `start_work` rejected (`ErrAgentHasActiveWork`); the slot never exceeds 1.
4. **Deployed DB structural guarantee** ‚Äî on a real install (SELECT-only, teardown), schema-version = 53 applied and the partial-unique index is enforced at the DB layer:
   ```sql
   CREATE UNIQUE INDEX idx_awi_agent_active

   ```
   ‚Üí even if application logic races, the DB rejects a 2nd active WorkItem. **By-construction, not by-convention.**

### 2.4 Notable verify-not-trust catches that shaped the backend

- **State-machine simplify (A) is safe** ‚Äî independently confirmed the task events are *already* generic (`EvtTaskStateChanged`, no per-action events) and dispatch hangs on `EvtTaskAssigned` (not status), so the single-generalized-endpoint design loses no events.
- **Free-model task‚ÜîWI decoupling is INTENDED, not a (b)-limbo bug** ‚Äî under SetStatus, a task can be "running" with its WorkItem "done" (status = agent self-reported label, center doesn't enforce the WI); the WorkItemProjector `EvtTaskStateChanged` handler is graceful for any target (specific cases + default no-op). Distinct from the (b) WI-failed limbo, where the system-projector MUST converge task‚ÜíBlock. Both faces classify by this distinction.
- **Tags 16-char limit is RUNE-based** (`len([]rune(tag)) > 16`) = CJK-correct (a 16-Chinese-char tag = 48 bytes is accepted). Propagated cross-layer: the FE chip input now counts by code-point (`[...tag].length`), and a CJK boundary test was added to the backend.
- **Batch PATCH is atomic all-or-none** (single tx; one invalid field ‚Üí whole rollback, no partial-apply, no version bump) ‚Äî confirmed by `TestBatchUpdateTask_Atomic`.
