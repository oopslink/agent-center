## 3. 缺口 / bug 清单 —— 哪些坏/缺(交 dev2) ❌

**run-real 实测确认的 4 条 polish 缺口(对全部会话类型一致,因共用同一 composer):**

| # | 缺口 | 实测证据 | 影响 |
|---|---|---|---|
| **G1** | **无拖拽上传(drag & drop)** | composer DOM 无 drop-zone、无 `ondrop/ondragover` handler | 终验硬门列项,当前只能点回形针选文件 |
| **G2** | **无粘贴截图(paste screenshot)** | `composer-textarea` 无 `onpaste` handler | 终验硬门列项,截图无法粘贴入框 |
| **G3** | **无上传进度(progress)** | 无 `progressbar`/进度元素;三段上传期间仅 `send.isPending` 禁用,无百分比/状态 | 大文件无反馈 |
| **G4** | **无类型/大小校验** | 文件 `<input>` 无 `accept=` 属性(任意类型放行);后端亦未见显式 size 上限 | 终验硬门列项;无客户端类型过滤、无大小上限护栏 |

**低优注记(非用户面 bug,记给写 API 级用例的人):**
- **N1** issue/task 会话**惰性创建**:刚经 `/api` 建好 issue/task 后立即 `GET /conversations?owner_ref=pm://{issues,tasks}/{id}` 返回 **0 条**;但 UI 进入详情页后 composer 正常渲染、发送成功(收端✅)。即会话在首次 UI 访问/发送时按需创建,API-only 消费者勿假设其 eager 存在。用户面无感知、不阻塞。

**未发现的问题(主动声明已验、未触发):** 发送/接收/下载/越权 全链路无报错、无控制台错误;图片内嵌预览与文件卡片元数据(类型标签 + `formatBytes` 大小)渲染正确;明暗双模附件卡片均可读。
