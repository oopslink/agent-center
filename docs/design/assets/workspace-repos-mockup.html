模型要点（待讨论确认 → 再建 plan）

  • Repo = workspace 顶级实体(侧栏与 Projects 并列):{label,description,url,provider,default_branch,credential(加密)}。凭据只在此配。
  • Project 引用:沿用现有 CodeRepoRef 作引用(无需数据迁移);additive 加可空 repo_id 指向 workspace Repo。primary 在引用层(每项目选主仓)。
  • 解析:merge-check / agent list_project_repos / viewing 都经 项目引用 → workspace Repo。凭据永不出 API/返给 agent。
  • viewing:go-github + git ls-remote 回退,不 clone。agent 接口:默认静态 + live 可选。
  • 权限:workspace Repo+凭据 = workspace/org admin;项目成员只能引用/取消引用。
