设计要点（详见 issue 描述 / docs/design/features）

  • 传输:Web Console → Center admin API → 下发 agent.runtime_fs{req_id,op,path} 命令 → Worker 读+脱敏 → POST .../runtime-fs/response{req_id} → Center 关联返回。**worker 必须在线**,离线/超时→unavailable。复用 control-loop + feedback,不开 Worker 监听端口。
  • 安全红线:mcp_config.runtime.json(明文凭据)**列出但内容不返回**(redacted);.git/ 隐藏;*.sock/*.lock 只 metadata。路径安全复用 resolveContainedPath(symlink/`..`/绝对路径防逃逸)。
  • 鉴权:org 维度——caller 是该 agent 的 org 成员。限制:目录 entries 上限 / 单文件预览 1MB / 超时。
  • memory:memory/ 是 git 仓 → v1 含 commit 历史(只读 git log)+ 文件预览;diff phase-2。
  • v1 排除:jsonl 结构化渲染 / markdown 渲染 / tail-SSE / 下载 / 任何写操作。
