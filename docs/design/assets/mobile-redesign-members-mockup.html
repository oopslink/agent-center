设计要点

  • 顶部 Humans/Agents 分段控件保留了 PC 端已有的 MembersSegmentControl 模式（唯一已存在的移动端组件），但整个视觉重做以贴合本批的设计语言；这是有意"保留交互形状、重画视觉"，不是照搬代码。
  • Agents 卡片补齐了现有移动端卡片丢掉的字段：角色、负载/待处理 chip、CLI/Model、最近活动——但不是所有 12 列都塞进卡片，而是按"识别 agent 状态最需要看什么"重新分层（生命周期徽章 > 负载 > 配置摘要）。
  • AgentDetail 头部生命周期按钮走横滑胶囊行，按钮集合随状态变化（图中是 running 态：发消息/Stop/Restart；stopped/error 态会换成 Start/Reset/Archive）——这套状态机较复杂，spec 里会用状态表逐条定义，不是本 mockup 能穷举的。
  • Reset 弹层完整保留"scope 单选 + 二次确认勾选未选中前提交按钮置灰"这个安全设计，不简化成一个按钮。
  • Runtime/Analytics 两个 tab 做了简化但不阉割的处理：文件树浏览保留只读预览+敏感文件屏蔽+离线态；Analytics 保留概览数字+热力图+Top花费任务，但去掉了桌面版更细的按模型/按项目切换视图（留后续）。这是"该做减法的地方做减法，但不是整个 tab 消失"。
  • MembersHumans 卡片同样补回 PC 桌面表格独有的 Joined/Disabled 状态徽章和"⋯"操作入口（变更角色/禁用/重新启用），现有移动端卡片完全没有这些。

功能覆盖清单（未逐屏画出，写 spec 时逐项定案）

  • Agents 列表批量多选 + 批量生命周期操作（现有移动卡片完全没有）：这是桌面独有的高强度运维操作，移动端是否需要对应的"长按多选"模式，需要显式决策而不是默默丢弃。
  • Agent 创建弹层（AgentCreateModal：名称/描述/model/CLI/Worker 选择）、Agent 配置编辑弹层（AgentConfigEditModal：并发/env vars/executor 白名单等）：本批只标了入口，表单细节留后续。
  • Force delete（GitHub 风格输入名称二次确认）：本批未画，但必须保留同等强度的安全确认，不能降级成普通 Yes/No。
  • Archive 操作：软删除，released worker、保留历史——留后续画确认弹层文案。
  • SenderDetailSidebar（点击消息里的 agent 头像弹出的精简预览）与 AgentDetail 的关系：前者是精简版，不能被误当作 AgentDetail 的替代品，两者都要保留。
  • Reminders 联动（SenderDetailSidebar 里 agent 专属的"创建提醒"，会扫描限流重置时间预填表单）：跨模块功能，留给 Reminders 批次一并考虑。
  • MembersHumans 页面本身没有邀请入口（邀请流程在 OrganizationSettings/MemberNew）：确认这一点，移动端不要在 MembersHumans 页面上误加"邀请"按钮。
