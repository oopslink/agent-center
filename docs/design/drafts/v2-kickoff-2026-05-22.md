## § 1. 候选总览（11 项，分 4 组）

按主题分 4 组。每组内的项之间多数独立，跨组有依赖关系（见 § 2）。

### Group 1｜算力弹性（E）

> 把"算力节点"做成显式一等公民，让 agent-center 能挂多台开发机和云节点，agent 不再绑死在单一 Worker daemon 上。

| # | 候选 | 来源 | 状态（2026-05-22）|
|---|---|---|---|
| **E1** | **Worker enroll 轻量化** —— 把 bootstrap token 流程做轻，加节点像 `slock` 一键秒级 | 竞品报告 § 3.6 + § 5.3 | ✅ 进 v2，[ADR-0023 草案](../decisions/drafts/0023-worker-enroll-lightweight.md) 已立 |
| **E2** | **云 Computer 节点支持** —— 注册云算力节点进算力池，按需拉起 agent | 竞品报告 § 3.6 | ⏸ 移入 [roadmap.md § v3 中期](../roadmap.md) |

> 所有跟 agent 实体 / 协议 / 能力相关的事都在这里看，不必跨组追查。包括把 "agent" 从隐式概念升级为聚合实体，以及 dispatch envelope 层的能力扩展。

| # | 候选 | 来源 | 状态（2026-05-22）|
|---|---|---|---|

## § 2. 跨组依赖关系

```
E1 (Worker enroll 轻量化) ──┬──> E2 (云 Computer 节点)
                            │
                            └──> G1 (AgentInstance) ──> G2 (agent:create 协议)
                                              │
                                              └──> G4 (MCP per-agent — 走 AgentInstance.config)

G3 (多 adapter)          ── 独立，可单跑
G5 (Skill file-mount)    ── 独立，可单跑

S1 (Reminder)            ── 独立小改动
S2 (Cron 触发器)         ── 独立
S3 (Emoji 反馈)          ── 独立

M1 (任务管理面)          ── 独立
```

**关键耦合**：
- **E1 是 G 组多数项的前置门槛** —— 想把 agent 从 Worker 解耦，得先让 Worker 节点变可挪。
- **G1 是 G2 / G4 的前置** —— `agent:create` 协议和 per-agent MCP 配置都需要 AgentInstance 实体存在。
- 其余项可并发或单独推进。

## § 3. 主线选项（待决）

下一步排具体顺序，有几种走法：

| 选项 | 描述 | 优缺点 |
|---|---|---|
| **a) 自然演进** | 按 E → G → S → M 顺序定 v2 主线 | 路线清晰但慢；E 组改动深 |
| **b) Quick wins** | 挑独立 + 低成本 + 立即可用的项先做：**G3 多 adapter + S1 reminder + S3 emoji 反馈** | 立竿见影但对 v2 没叙事；属于 v1.x 增量发布性质 |
| **c) Thesis 推进**（**Agent 倾向推荐**） | 走 **E1 → G1 → G2**，把 AgentInstance + 动态 spawn 一起做掉 —— 这是改 Workforce BC 的最大一笔，让 agent-center "长出第二条腿"（多 worker 节点 + agent 跨节点） | 路线最有 v2 叙事；改造工作量最大；需要立 ADR-0022 + 可能 superseded ADR-0008 |
| **d) 目标倒推** | 用户给出 v2 版本目标（想干什么 / 期望多久出活），按目标排候选项 | 最贴用户实际意图；要先有目标陈述 |

## § 4. 已明确的红线（v2+ 也不做）

> 来自 [`docs/design/architecture/strategic/00-domain-vision.md`](../architecture/strategic/00-domain-vision.md) 边界 + 竞品报告 § 5.1。

- ❌ **多租户 / SaaS** —— Vision B2 总开关；走那条等于重做项目
- ❌ **协作中枢级 web 看板** —— Multica Linear/Jira 全家桶规模；4 团队级 UI 复杂度；任务管理面 ≠ 协作中枢，详见 § 5.1 修订
- ❌ **Agent polymorphic actor**（agent 跟 human 平权造任务）—— 拆 S4 thesis（详见竞品报告 § 5.1 子项详解）
- ❌ **多 IM vendor** —— v1 锁飞书，Bridge BC 保持 ACL 抽象但不在 v2 做多 vendor
