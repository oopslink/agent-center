# v2.5 → v2.6 Migration Guide

v2.6 introduces the **Identity BC**: multi-tenant organization management, full
auth (signup/signin/signout), and org-scoped URL routing. It also removes the
Supervisor concept from the Cognition BC.

## Prerequisites

- You are running Agent Center v2.5.x.
- You have shell access to the machine running `agent-center`.

---

## Step 1: Stop the current service

```bash
# macOS
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/agent-center.plist

# Linux (systemd)
systemctl --user stop agent-center-center
```

---

## Step 2: Install v2.6

Download the v2.6 binary and replace the existing one via the standard
install mechanism, or run:

```bash
agent-center install center --version v2.6.0
```

The installer performs a symlink swap and restarts the service. If you are
upgrading manually, replace the binary at `/opt/agent-center/bin/agent-center`
and restart.

---

## Step 3: Schema migration

v2.6 adds new BC9 Identity tables (migrations 0033–0036) and drops the old
`supervisor_invocations` and `decision_records` tables. The schema migration
runs automatically at server start via `persistence.NewMigrator(db).Up(ctx)`.

**No manual SQL is required.**

If you want to apply the migration manually before starting the server:

```bash
agent-center migrate v1-to-v2 --apply --archive-dir /tmp/v2-archive
```

This is only needed when upgrading from v1 (schema version < 32). For v2.5.x
installations (schema version 32–35), the server applies migrations 0033–0036
automatically at startup.

---

## Step 4: First signup

On first boot after the upgrade, the browser will redirect to `/signup`.

1. Enter your **display name** (1–40 characters).
2. Enter a **6-digit numeric passcode** and confirm it.
3. Enter your **organization name** and **organization slug** (3–40 chars,
   `[a-z0-9-]` only).
4. Submit. You will be logged in and redirected to `/organizations/{slug}/`.

> **Note:** The old `identity.default_user` config key is removed. All
> identity management is now handled by the Identity BC through the web UI.

---

## Step 5: Verify

- Open `http://localhost:8080/` — should redirect to `/signup` (fresh) or
  `/organizations/{slug}/` (already signed up).
- Navigate to `/organizations/{slug}/fleet` to verify workers and tasks.
- Navigate to `/organizations/{slug}/members/humans` to see your account.

---

## What changed for users

### Bookmark / URL changes

All existing deep-link URLs now have an org prefix:

| Old URL | New URL |
|---------|---------|
| `/channels/my-channel` | `/organizations/{slug}/channels/my-channel` |
| `/projects` | `/organizations/{slug}/projects` |
| `/fleet` | `/organizations/{slug}/fleet` |
| `/tasks/{id}` | `/organizations/{slug}/tasks/{id}` |

Existing bookmarks without the org prefix will auto-redirect to the first org.

### Removed features

- **Supervisor commands:** `agent-center supervisor ...` CLI commands are
  removed. Supervisor-related event types (`supervisor.invocation.*`,
  `supervisor.decision.*`) are no longer emitted.
- **Admin identity endpoints:** `GET/POST /admin/identity/...` admin API
  endpoints are removed. Use the web UI (`/members/humans`) instead.
- **Conversation identity subdomain:** internal `conversation/identity`
  package removed. Identity validation uses `IdentityBCFacade` instead.

---

## Worker daemons

Worker daemons do not require any changes. They authenticate via admin tokens
(unchanged from v2.5). Re-enroll tokens continue to work.

---

## Configuration changes

| Config key | Change |
|---|---|
| `identity.default_user` | **Removed.** |
| `secret_management.master_key_file` | Now also used as JWT signing key. No change required. |

---

## Rollback

v2.6 is a forward-only upgrade due to the Identity BC table additions. To
roll back:

1. Restore the database from a backup taken before the upgrade.
2. Reinstall the v2.5 binary.

There is no automated downgrade path. The migration 0036 `down` script
restores `supervisor_invocations` and `decision_records` for round-trip
testing only — it does not restore the v2.5 schema in full.
