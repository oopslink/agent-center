Web Console 使用指南

agent-center 的浏览器操作界面 -- 嵌入 server 单一二进制，loopback-only 访问。覆盖界面总览、任务与议题、Plan 编排、消息、Input Request 和 Work Board。

§ 1 界面总览

1.1 三栏式 Shell

Web Console 采用三栏式布局（v2.8.1 redesign），由左到右：

栏位置内容
Module Rail（侧边栏）左侧固定组织切换器、页面导航（Channels / DMs / Tasks / Issues / Plans / Fleet / Agents / Secrets / Settings）、Command Palette 触发（Cmd+K）、主题切换、SSE 状态指示、用户信息
List / Detail（主区域）中央当前页面的列表视图或详情视图 -- 如 Channel 列表、Task 详情 timeline、Plan DAG 图等
Context Sidebar（上下文面板）右侧可选选中实体的元数据摘要（如 Task 状态 / 分配人 / 关联 issue / execution trace）、参与者列表、conversation sidebar

1.2 键盘快捷键

快捷键功能
Cmd+K打开 Command Palette 全局搜索
Cmd+B折叠 / 展开侧边栏
Cmd+D切换 Light / Dark 主题
Cmd+1..7跳转到对应的顶级页面

1.3 实时更新（SSE）

Web Console 通过 /api/sse 单连接接收 server 端实时事件。所有列表页面在收到相关事件后自动刷新数据 -- 不需要手动 reload。

侧边栏顶部的 SSE Indicator 圆点显示连接状态：绿色 = 在线，灰色 = 断线重连中。断线时页面仍可用（显示 react-query 缓存数据），重连后自动追赶。

SSE 断线重连策略：base=1s, max=30s, jitter ±20%。30 秒无任何事件（含心跳）则强制重连（Safari / iOS half-open 保护）。

1.4 访问方式

本机直接打开 http://127.0.0.1:7100；远程通过 SSH tunnel：

ssh -L 7100:127.0.0.1:7100 user@center-host
# 然后浏览器打开 http://127.0.0.1:7100/

§ 2 Tasks 与 Issues

2.1 创建 Task

创建方式有三种：

  1. 从消息派生 -- 在 Channel / DM / Issue 对话中，点击 "Select messages" 进入多选模式，选中消息后点击 "Derive Task"，选择目标 Project 并确认。被引用的消息会以 carry-over 形式出现在 Task 的 thread 中。
  2. 直接创建 -- 在 Tasks 列表页或 Project 详情页点击 "+ New Task"，填写标题、描述、所属 Project。
  3. 从 Work Board 创建 -- 在 Board 视图的任意列点击 "+" 快速创建。

2.2 Task 状态生命周期

状态含义流转
submitted已创建，等待分配→ running / canceled
runningAgent 正在执行→ paused / completed / failed
pausedAgent 暂停（如等待 Input Request）→ running / canceled
completed执行成功→ reopened
failed执行失败→ reopened
reopened已完成/失败的 task 被重新打开→ running
canceled已取消终态

2.3 分配给 Agent

Task 创建后可在详情页右侧 sidebar 的 Assignee 区域将其分配给一个 Agent。分配后 worker daemon 会自动拉取 dispatch envelope 并启动 agent CLI 执行。也可以在 Plan 编排中批量设置 assignee。

2.4 创建 Issue

Issue 是持续跟踪的议题，与 Task（一次性执行）互补。在 Issues 列表点击 "+ New Issue"，或从消息多选后点击 "Open Issue" 派生。Issue 有自己的 discussion thread，支持状态 filter（open / closed）。

2.5 从 Issue 派生 Task

在 Issue 详情的 discussion 中，同样可以多选消息 → "Derive Task"。该 Task 自动关联 parent Issue，在 Issue 详情的 sidebar 中可看到所有关联的 Tasks。

§ 3 Plan 编排

3.1 创建 Plan

Plan 用于编排多个 Task 的执行顺序（DAG 依赖图）。在 Project 详情页的 Plans tab 点击 "+ New Plan"，输入名称即可。新建的 Plan 处于 draft 状态。

3.2 添加 Tasks

在 Plan 详情页可以将 Project 下的 Task 加入 Plan。每个加入 Plan 的 Task 成为一个 Plan Node，在 DAG 视图中显示为节点。

3.3 设置 DAG 依赖

Plan 在 draft 状态时，DAG 依赖可编辑。在 Plan 详情页的 DAG tab 中：

  • 添加依赖 -- 使用节点下方的 dependency selector，选择 "from → to" 节点对来创建边。
  • 移除依赖 -- 点击边列表中的 "Remove" 按钮。
  • 移除节点 -- 从 Plan 中移除一个 Task node。
后端自动校验：禁止自环（self-dependency）和成环（cycle）；running / done 状态的 Plan 不允许编辑依赖。

3.4 启动与自动调度

Plan 编排完成后，点击 "Start Plan"。Orchestrator 自动根据 DAG 拓扑序调度：

  1. 无依赖（入度为 0）的 Tasks 立即派发给 agent 执行
  2. 前置 Tasks 全部 completed 后，后续 Tasks 自动解锁并派发
  3. 任一 Task failed 时，Plan 暂停并标记 failed indicator
  4. 可在 Plan 详情页点击 "Advance" 手动推进暂停的节点，或 "Resume" 恢复 paused node

3.5 Plan 状态

状态含义
draft编辑中，可增删 node 和 dependency
running执行中，DAG 自动推进
done所有 node 完成

3.6 Plan 详情三 Tab

Plan 详情页提供三个 tab：

  • Chat -- Plan conversation thread，可在此讨论编排策略
  • DAG -- 拓扑图可视化，每个节点显示 Task 名称 + 派生状态（pending / running / done / failed）
  • Tasks -- 平铺的 Task 列表，可快速查看状态和 assignee

§ 4 消息与文件

4.1 对话类型

类型路径说明
Channel/channels/:name团队频道，可添加人类成员和 agent 参与者
DM/dms/:id一对一私信，支持 peer + agent quick-pick
Task thread/tasks/:idTask 详情内嵌的对话 thread
Issue thread/issues/:idIssue 详情内的 discussion
Plan thread/projects/:id/plans/:planIdPlan 详情的 Chat tab

4.2 发送消息

在任意对话底部的 Composer 输入内容：

  • Enter -- 发送消息
  • Shift+Enter -- 换行
  • 发送失败时错误会 inline 显示，输入内容不会丢失，可直接重试

4.3 @mention

输入 @ 后出现 autocomplete 列表，可 mention 人类用户或 agent。被 mention 的实体会收到对应事件通知。

4.4 文件上传

消息 composer 支持附件上传。文件存入 BlobStore，在 timeline 中以链接形式展示。

4.5 Carry-over 消息

当从 Channel / Issue 的消息派生 Task 或 Issue 时，被选中的消息以 carry-over 形式引入目标 thread。Carry-over 消息有特殊的 Divider 标注来源 conversation，并以 "Discussion below" 分隔原始引用和后续讨论。

§ 5 Input Request

5.1 Agent 升级请求

当 agent 在执行 Task 过程中遇到需要人类决策的节点，会发起 Input Request（升级请求）。此时：

  • Task 状态可能切换到 paused（agent 主动等待）
  • Web Console 导航的 Input Request badge 实时更新待回应数量
  • Fleet 页面的 "Open IRs" segment 显示当前所有活跃的 Input Requests

5.2 Input Request 收件箱

在 /inputrequests 页面查看所有待回应的 Input Request 列表。每条 IR 显示：

  • 发起的 agent 和所属 Task
  • 请求的问题描述
  • 创建时间

5.3 回应流程

  1. 点击一条 Input Request 打开 Respond Modal
  2. Modal 中显示 agent 提出的问题和可选的 option chips
  3. 输入回应内容或选择预设选项，点击 "Respond" 提交
  4. Agent 收到回应后继续执行，Task 状态恢复为 running

5.4 取消 Input Request

如果不需要回应，可以点击 "Cancel" 并在确认对话框中确认取消。取消后 agent 会收到取消信号。

5.5 Adopt suggestion

部分 Input Request 会携带 agent 建议的选项，Respond Modal 中以 chip 形式展示，点击即可快速采纳建议。

§ 6 Work Board

6.1 Kanban 视图

Work Board 提供看板（Kanban）视图，将 Task 按状态分列展示。每列对应一个 Task 状态（submitted / running / paused / completed / failed）。

6.2 操作

  • 拖拽重分配 -- 将 Task 卡片从一列拖到另一列以更改状态。卡片内会显示 Task 标题、分配的 agent、所属 Project。
  • 快速创建 -- 每列顶部有 "+" 按钮，点击后弹出 BoardTaskCreateModal 快速创建新 Task 到该状态列。
  • 筛选 -- Work Board 支持按 Project、Assignee、状态筛选卡片。

6.3 移动端适配

移动端的 Work Board 切换为纵向堆叠布局，支持触摸拖拽（useBoardTouchDrag）。

6.4 Org 级 Work Items

在 Organization 层级，/organizations/:slug/issues 和 /organizations/:slug/tasks 页面提供跨 Project 的 Issue / Task 聚合视图，支持：

  • 多状态 filter（multi-select）
  • 多 Project filter
  • Assignee filter（人类或 agent）
  • 创建 / 更新时间范围 filter
  • 服务端排序 + 分页
