MCP 服务 / Agent 接口

每个 agent 子进程通过一个 per-agent 的 MCP 服务（mcp-host）调用 center 的能力——领取/回报工作、读写任务与 Issue、编排 Plan、收发会话消息、传文件、记录 findings。下方接口清单由代码生成（make gen-mcp-docs 从 internal/mcphost 导出），改工具后复跑即更新，绝不手写。

概述 · Overview

是什么

给 agent LLM 用的工具服务。每个 agent 子进程绑定一个 mcp-host 进程，它把模型发起的 MCP 工具调用桥接到 center 的 admin agent-tool 接口（/admin/agent-tools/<tool>）+ 文件搬运 seam。一进程只服务一个 agent。

怎么连

传输为 stdio（worker 以子进程拉起 claude，注入该 MCP server）。鉴权走 worker bearer token（worker:<id>）；操作 agent 身份是进程固定的（AC_MCP_AGENT_ID），注入每个调用体，不取自模型参数——模型无法假冒他人。center 每次再校验 requireAgentOnWorker + 域授权。

在拓扑里的位置

位于 Worker → Agent 之间：Worker spawn agent CLI，agent 经 MCP 回连 center 取工具。详见 部署架构 · 等距拓扑图 里的 Agent → Center 链路。

📌 接入 / 起步：生产环境由 worker 自动拉起并注入 MCP server，无需手动配置；本页是接口 reference。如何安装 worker、派单、起 agent，见手册 CLI & MCP 章。工具总数与分组以代码为准——本页清单由 make gen-mcp-docs 生成。
