存储架构

所有持久状态的物理布局 —— 单 SQLite 库 + BlobStore + 事件表 + Memory(git)，跨 BC 不共享物理表。

SQLite 单库agent-center.db · 每个 BC 自己的表
单库进程模型
只有 server 进程开库；handlers 不直接开 sqlite。CLI / worker / web 都经 admin transport。
跨 BC 边界
跨 BC 经事件 / 引用，不共享物理表——每个 BC 拥有自己的表集合。
↓ 状态变更与事件同事务双写
事件存储events 表 · append-onlyADR-0014
events
append-only、单调 seq；与状态变更同事务双写（EventSink.Emit，调用方持事务）。
↓ 文件 / blob 与 trace 归档
BlobStore (文件)ULID 寻址 · 引用计数 GC
blob
ULID 寻址 blob，FileReference 按 scope 放置。
引用计数 GC
零活引用过宽限期回收。
FileTransferSession
上传 / 下载会话（open→completed）。

Trace：AgentTraceEvent → 本地 trace.jsonl → execution 结束归档进 BlobStore（不入 events 表，ADR-0015）。

↓ 记忆不落库
Memory (记忆)不在 DB · git 仓 · GitOps
scoped CLAUDE.md
不在 DB——git 仓里的 scoped CLAUDE.md 文件（$AGENT_CENTER_MEMORY_DIR），GitOps init/commit。
