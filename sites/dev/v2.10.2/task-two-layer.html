Task / TaskExecution 两层模型

为什么任务的「身份」和「每一次运行」要拆成两个独立聚合根 —— 这是 agent-center 调度可靠性的根。

TL;DR  Task 是不变的工作单元身份；TaskExecution 是它的每一次 dispatch。retry 不新建 task，只追加一条 execution。一图胜千言 → 回 架构浏览器看类图与序列图。

为什么两层

单层模型里，「重试一个失败任务」要么覆盖原记录（丢历史），要么复制成新任务（身份漂移）。两层模型把身份与运行痕迹分开：Task 持有稳定身份，TaskExecution 记录每一次从 dispatch 到结束的过程。

  • Task 1 : N TaskExecution —— 一个任务可被多次执行。
  • 每条 TaskExecution 永远绑定一个 worker，由 A2A 状态机驱动。
  • 只有 center 能创建 task，无野任务（conventions §1）。

Task：身份不变

Task 是独立 Aggregate Root，属于一个 project，并跟一条 kind=task 的 Conversation 1:1 绑定（task.conversation_id）—— 对话即决策审计。

TaskExecution：每一次执行

TaskExecution 持 task_id 强引用 Task。它的子实体包括卡住时的 InputRequest（同步阻塞请示）和产出的 Artifact（PR URL / 文件 / 报告，free-form kind）。

# retry = 同 task 新 execution，不创建新 task
Task task_01H… ─┬─ TaskExecution #1 → failed (timeout)
              ├─ TaskExecution #2 → killed
              └─ TaskExecution #3 → completed ✓ → Task done

状态机

层状态
Taskopen · suspended · done · abandoned
TaskExecutionsubmitted · working · input_required · completed · failed · killed

retry 语义

当某条 execution 走到 completed 终态，Task 自动联动为 done。失败可重试 —— 新 execution、同 task，历史完整保留。
