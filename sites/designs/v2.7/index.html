agent-center v2.7 · DDD 设计

← 设计版本索引 领域重构：项目管理 · 智能体 · 环境 · 会话 · 可观测性

v2.7 把系统重新划分为 5 个限界上下文。四个核心域承载业务真相（项目管理、智能体、环境、会话），可观测性作为只读支撑域横向跨读各域事件，身份作为共享内核提供统一的身份引用词汇。旧的「任务执行域 / 讨论域 / 工作力-项目」已退役。

U/D 上游/下游（信息流方向） OHS 开放主机服务 ACL 防腐层 CF 遵从者 SK 共享内核

上下文划分

核心域居中加宽（span 2），支撑域与通用域环绕。

项目管理 ProjectManager

核心域 CORE

工作管理的真相源：项目、议题、任务及其订阅与状态流转。

  • 聚合：Project 项目 / Issue 议题 / Task 任务 / ProjectMember 成员
  • 订阅真相（creator/assignee/manual），会话参与者据此投影

智能体 Agent

核心域 CORE

长驻智能体的生命周期、工作项分派与活动流。

  • 聚合：Agent 智能体 / AgentWorkItem 工作项 / ActivityEvent 活动事件

环境 Environment

核心域 CORE

工作者部署、进程控制、控制信道与文件传输。

  • 聚合：Worker 工作者 / FileTransferSession 传输会话 / BlobStore
  • AgentController 控制信道（worker 主动、offset+幂等双层去重）

会话 Conversation

核心域 CORE

统一的消息流与参与者；任务/议题各绑定一条会话。

  • 聚合：Conversation 会话 / Message 消息（追加写）
  • kind：dm / channel / adhoc / notification / task / issue

可观测性 Observability

支撑域 · 只读

机队监控、跨域事件查询、读模型投影、统计与日志。

  • 聚合：Event 事件（追加写）
  • 读侧：机队 fleet / 检视 inspect / 查询 query / 统计 / 日志

身份 Identity

通用域 · 共享内核

统一身份引用词汇，供所有域做跨域引用。

  • IdentityRef：user:<id> / agent:<id> / system

集成关系

各核心域作为上游把领域事件发往可观测性（下游）；其余为引用/投影/共享内核关系。
项目管理→U/D→可观测性 发 task.* / issue.* 事件入事件流
智能体→U/D→可观测性 发 agent.* / work_item.* 事件
环境→U/D→可观测性 发 worker.* / environment.* 事件
会话→U/D→可观测性 发 conversation.* 事件
可观测性→OHS→前端 / 运维 机队/检视/查询 只读 API（开放主机服务）
智能体→ACL→项目管理 工作项以 pm://tasks/{id} 非拥有引用任务
智能体→D→环境 读 Worker 状态派生 Agent.availability（环境优先）
会话→CF→项目管理 参与者镜像订阅真相（遵从订阅模型）
身份→SK→全部域 IdentityRef 作为跨域共享内核词汇

逐个核心域拆解聚合根（AR）、实体、值对象（虚线框）、领域服务与 Repository，状态机标注非终态→终态。橙色标签为该聚合发出的领域事件（过去式）。

Actor 触发者 Command 命令 Aggregate 聚合 Event 领域事件 Policy 策略

🆕 创建并派发任务（项目管理 → 智能体 → 环境）

用户（控制台/CLI）→ task:create→ Task 任务→ task.created
策略：自动派发→ enqueue_work_item→ AgentWorkItem 工作项→ work_item.queued
AgentController 控制信道→ activate_work_item→ Agent + WorkItem→ agent.activatedwork_item.activated
智能体→ post_task_message→ Message 消息→ conversation.message_posted
仅显式发布的消息进会话；冗长中间过程落 AgentActivityEvent 观测流。

✅ 完成与验收（智能体 → 项目管理 → 会话）

智能体→ complete_task→ Task→ task.completedwork_item.done
策略：待验收→ 用户 / 同侪 Agent→ verify_task→ Task→ task.verified
会话系统→ post_system_message→ Message→ conversation.message_posted
约束：验收者 ≠ 完成者（禁止自验，支持同侪评审）。

💬 议题生命周期 → 孵化任务（会话 → 项目管理）

用户（议题会话中）→ derive_issue→ Issue + Conversation→ issue.openedconversation.openedcarry_over_created
用户→ conclude_issue(spawn_tasks)→ Issue + Task[N]→ issue.status_changed→closed_with_taskstask.created ×N
议题 ↔ 会话 1:1 不可变绑定；子任务会话经 carry-over 继承议题会话上下文。

📁 文件上传与引用（环境 → 会话 → 可观测性）

智能体 / 用户→ initiate_upload(scope=task)→ FileTransferSession + BlobStore→ file_upload_initiated
Worker 守护进程→ complete_upload→ FileTransferSession→ file_upload_completed
策略：附件入会话→ post_message_with_attachment→ Message + FileReference→ conversation.message_postedfile_referenced
一 blob 多 scope 引用；任务删除软删引用，ref_count=0 + 7 天宽限后回收 blob。

💓 Worker 心跳与 Agent 可用性（环境 → 智能体）

Worker 守护进程→ heartbeat→ Worker→ worker.online
策略：派生可用性（环境优先）→ Agent（availability 投影）→ agent.availability_changed
心跳超时→ Worker→ worker.offline{reason}→ 级联→ agent → unavailable
Agent.availability 是只读投影（不落库）；Worker 状态优先。
