agent-center 是什么

agent-center 是一个个人 AI 智能体调度中心：你跑一个 server，从任意机器接入 worker，智能体在任意位置运行，而每一次派单、每一次输入请求、每一条决策与产物，都落在一个可追溯的会话线程上。

它的核心论点是 —— 会话（Conversation）是产品的脊梁，而不只是日志。Task、Issue、决策、进度全部挂在会话线程上；每一次 dispatch、每一个 InputRequest、每一份 Artifact 都能在它原本的线程里被找回。

实现上，agent-center 是一个多智能体编排平台：Go 后端 + React/Vite 编写的 Web Console，通过 go:embed 打包进同一个二进制分发。

解决什么问题

当多个智能体跑在多台机器上时，常见的痛点与 agent-center 的应对方式：

痛点agent-center 如何应对
多个智能体散落在多台机器、状态分散在 N 个终端里 一个 server 汇聚一切；/fleet 实时呈现每个 worker × 执行 × 待处理 IR
智能体执行到一半停下来问你（"要不要提交？"） InputRequest 是一等概念 —— 在 Web Console 的卡片里作答，智能体随即恢复
很难追溯智能体做了什么、为什么做、奉谁之命 每个 Task / Issue 都有一条会话线程；派单、决策、进度与产物全部落在其中
Skill / MCP 配置散落在每个智能体各自的仓库里 AgentInstance 是一等聚合根：指令 + MCP servers + skill 挂载都绑定到智能体身份
凭据管理 UserSecret 限界上下文，AES-256，明文永不回显；智能体以 secret:<name> 引用密钥
多机部署 多主机 TCP+TLS（SSH 式指纹固定）+ 一条命令完成首跳接入
核心能力 / 特性

一中心，多 worker，任意位置运行

跑一个 server，从任意机器接入 worker。智能体在任意位置运行 —— 每一次对话与决策都落在一个你能回溯的线程上。

装好即用

一条命令装好中心，一条命令装好每个 worker；一对配套的 upgrade 命令向前推进既有安装（原子 symlink 切换 + 健康探针 + 自动回滚）。

会话是脊梁

每一次派单、每一个 InputRequest、每一份产物，都能在它原本的会话线程里被找回。

InputRequest 一等公民

智能体执行到一半阻塞、向你请求决策；你在 Web Console 作答，智能体随即恢复执行。

Fleet 实时视图

/fleet 实时呈现每一个 worker × 执行 × 待处理的 InputRequest。

开箱即用的鉴权

订阅版 /login 开箱即用；智能体的 claude 复用同一份登录 —— 无需额外配置。

单机多 worker · 跨主机 TCP+TLS

同一台机器可跑多个 worker，并支持多主机 TCP+TLS 连接，采用 SSH 式的指纹固定（pinning）。

整体架构概览

单个 agent-center 二进制以多种模式运行。以下只描绘代码中真实存在的运行时组件。

中心服务 · agent-center server

汇聚全部状态，独占 SQLite 数据库，对外暴露以下端点：

Web Console

内嵌的 React SPA（go:embed），仅监听回环 127.0.0.1:7100。

Admin 端点

本地 unix socket，及/或供跨主机 worker 使用的 TCP+TLS :7300。

Center 监听

默认监听端口 :7050（避开被 macOS AirPlay 占用的 7000）。

▲ ▼  仅经由 admin 端点通信（worker 从不直接打开 SQLite）
Worker 守护进程 · agent-center worker run --config=…

运行在同一台或任意其他机器上的控制流执行器，负责拉起智能体 CLI。自 v2.7 起，worker 守护进程随统一的 agent-center 二进制一并分发。其下还包含：

MCP host

agent-center worker mcp-host —— 由守护进程为每个智能体拉起的 stdio MCP server，向智能体的 claude 进程暴露 agent-center 工具集。

Agent supervisor

agent-center worker agent-supervisor —— 每智能体一个的常驻 supervisor 进程（增量能力，尚未接入守护进程的拉起路径）。

Supervisor（领域概念）是一个内置的 AgentInstance：它读取会话上下文，决定下一步该派发什么 —— "不是一个'大脑'，只是另一个带日志的智能体"。
